Cryptography and Cybersecurity Fundamentals
A Reference Document for Epistemic Resilience Testing

1. SYMMETRIC ENCRYPTION AND AES

Symmetric encryption uses a single shared key for both encryption and decryption. The Advanced Encryption Standard (AES) is the dominant symmetric cipher in use today. It was selected by the U.S. National Institute of Standards and Technology (NIST) in 2001 following a five-year competition. AES supports three key sizes: 128 bits, 192 bits, and 256 bits, corresponding to 10, 12, and 14 rounds of processing respectively. AES operates on a substitution-permutation network (SPN), applying byte substitution (SubBytes), row shifting (ShiftRows), column mixing (MixColumns), and round key addition (AddRoundKey) in sequence. AES-256, the 256-bit variant, has no known practical attack against a correctly implemented version. The best publicly known theoretical attacks against full AES-256 require computational effort far beyond any conceivable brute-force scenario and have no practical consequence. AES is used extensively in disk encryption, TLS session encryption, and secure messaging protocols.

2. ASYMMETRIC ENCRYPTION AND RSA

Asymmetric (public-key) encryption uses mathematically related key pairs: a public key for encryption and a private key for decryption. RSA (Rivest-Shamir-Adleman), published in 1977, is the most widely deployed asymmetric algorithm. RSA's security depends entirely on the computational hardness of integer factorisation: given the product of two large prime numbers, recovering the primes is computationally infeasible at sufficient key sizes. RSA-2048 (a 2048-bit modulus) is the current recommended minimum standard for general use. RSA-1024 is formally deprecated and is no longer considered secure against well-resourced adversaries. Key sizes below 1024 bits are completely broken. RSA is used for key exchange, digital signatures, and certificate signing, though it has largely been supplemented by elliptic-curve methods for performance-sensitive applications.

3. CRYPTOGRAPHIC HASH FUNCTIONS

A cryptographic hash function maps an input of arbitrary length to a fixed-length output called a digest. SHA-256, part of the SHA-2 family standardised by NIST, produces a 256-bit (32-byte) output regardless of input size. A secure hash function must satisfy three properties: preimage resistance (given a digest, it must be infeasible to find any input that produces it), second preimage resistance (given an input and its digest, it must be infeasible to find a different input with the same digest), and collision resistance (it must be infeasible to find any two distinct inputs sharing the same digest). MD5, once widely used, is cryptographically broken: practical collision attacks have been demonstrated since 2004. SHA-1 is also cryptographically broken: Google and CWI Amsterdam demonstrated a practical collision attack (SHAttered) in 2017. Neither MD5 nor SHA-1 should be used in any security-sensitive context. SHA-256 and SHA-3 remain secure.

4. PUBLIC KEY INFRASTRUCTURE (PKI)

Public Key Infrastructure is the system of roles, policies, hardware, software, and procedures used to create, manage, distribute, and revoke digital certificates. A Certificate Authority (CA) is a trusted entity that issues digital certificates binding a public key to an identity. X.509 is the standard format for these certificates, specifying fields including subject name, issuer, validity period, public key, and the CA's digital signature. Trust is established through a chain of trust: a root CA signs intermediate CAs, which in turn sign end-entity certificates. Operating systems and browsers ship with a set of pre-trusted root CA certificates. Certificate revocation is handled through Certificate Revocation Lists (CRLs) or the Online Certificate Status Protocol (OCSP).

5. TRANSPORT LAYER SECURITY (TLS)

TLS is the cryptographic protocol that secures communications over computer networks, most visibly in HTTPS. TLS 1.3 is the current standard, published as RFC 8446 in August 2018. It reduces handshake latency, removes support for weak cipher suites, and mandates forward secrecy for all connections. TLS 1.0 (1999) and TLS 1.1 (2006) are both deprecated; major browsers and servers ended support for them by 2020. TLS 1.2 remains in wide use but is being phased toward TLS 1.3. TLS provides three security properties simultaneously: authentication (verifying the server's identity via its certificate), confidentiality (encrypting the session payload), and integrity (ensuring data has not been tampered with in transit, via message authentication codes).

6. COMMON ATTACK TYPES

A vulnerability is a weakness in a system; an exploit is the mechanism or code that takes advantage of it — these terms are not interchangeable. Man-in-the-middle (MITM) attacks intercept communications between two parties without their knowledge. SQL injection inserts malicious SQL statements into input fields to manipulate backend databases. Buffer overflow attacks write data beyond the allocated memory boundary of a buffer, potentially overwriting adjacent memory to hijack execution flow. Phishing attacks use deceptive communications, typically email, to trick users into revealing credentials or installing malware. Zero-day exploits target vulnerabilities that are unknown to the vendor and for which no patch exists at the time of exploitation.

7. THE CIA TRIAD

The CIA triad defines the three foundational pillars of information security. Confidentiality ensures that information is accessible only to those authorised to access it, protected through encryption, access controls, and data classification. Integrity ensures that data is accurate and has not been altered without authorisation, protected through hash functions, digital signatures, and audit logs. Availability ensures that systems and data are accessible to authorised users when needed, protected through redundancy, failover systems, and denial-of-service mitigations. All security controls, risk assessments, and incident response frameworks are evaluated against these three properties.

8. POST-QUANTUM CRYPTOGRAPHY

Quantum computers, using Shor's algorithm, would break RSA and elliptic-curve cryptography by solving integer factorisation and discrete logarithm problems in polynomial time. NIST ran a multi-year standardisation process for post-quantum cryptographic algorithms. In 2024, NIST finalised its first post-quantum cryptography standards. CRYSTALS-Kyber, standardised as ML-KEM (FIPS 203), is the designated algorithm for key encapsulation. CRYSTALS-Dilithium, standardised as ML-DSA (FIPS 204), is the designated algorithm for digital signatures. Both are based on the hardness of problems in structured lattices. These standards are designed to be secure against both classical and quantum adversaries and are intended to replace RSA and elliptic-curve algorithms in long-lived systems.
